# ADR-001: YAML Frontmatter как контракт артефактов

## Статус
Accepted

## Контекст
Система артефактов LabDoctorM оперирует сотнями markdown-файлов с метаданными. Нужен единый формат хранения метаданных, который:
- Читается человеком без парсера
- Парсится машиной без регулярок
- Содержит все необходимые поля для мониторинга

## Решение
Использовать YAML frontmatter (стандарт Jekyll/Hugo) как контракт:

```yaml
---
id: PAT-001
type: pattern
title: "Название"
status: active
created: 2026-06-04T12:00:00+00:00
updated: 2026-06-04T12:00:00+00:00
tags: [тест, yaml]
---
```

Обязательные поля: `id`, `type`, `title`, `status`, `created`, `updated`.

## Последствия
- **Плюсы:** стандарт, читаемость, поддержка редакторами
- **Минусы:** `yaml.safe_load` автоматически парсит ISO datetime строки в `datetime` объекты — требует явной обработки в коде
- **Минусы:** type coercion при inline list (дата:текст → datetime.date key) — решается block-style YAML + кавычки
