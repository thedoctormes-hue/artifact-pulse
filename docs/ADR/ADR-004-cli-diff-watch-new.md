# ADR-004: Новые CLI-инструменты (diff, watch, new)

## Статус
Принят

## Контекст
artifact-pulse v4.2.0 имел 11 entry points для анализа и мониторинга артефактов, но не имел инструментов для:
- Сравнения двух артефактов (исторический анализ изменений)
- Планирования health check (для systemd timer)
- Генерации новых артефактов с валидным frontmatter

## Решение
Добавлены 3 новых модуля с entry points:

### artifact-diff.py
Сравнение двух артефактов по ID. Показывает:
- Изменения frontmatter-полей (diff ключ-значение)
- Различия в исходящих ссылках (добавленные/удалённые)
- Различия во входящих ссылках
- Body diff (unified diff через difflib)

### artifact-watch.py
Планировщик health check. Режимы:
- `--once` — однократная проверка (для systemd timer)
- `--interval N` — цикл с интервалом в минутах
- Выводит только алерты; если алертов нет — молчит (exit 0)

### artifact-new.py
Генератор шаблонов артефактов. Автоматически:
- Определяет следующий свободный ID для типа (PAT-005, ADR-012, etc.)
- Генерирует валидный frontmatter (status=draft, confidence=high, source=agent)
- Записывает в правильную директорию

## Последствия
- 14 entry points (было 11)
- Тесты для каждого модуля (16 новых)
- Обратная совместимость сохранена
- Новые фичи не увеличивают runtime-зависимости (stdlib-only)
